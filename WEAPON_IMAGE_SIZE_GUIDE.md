# 背包武器图片尺寸规范

本文档给策划、美术和后续换皮 agent 使用，目标是让背包格子、下方备战区、拖拽中的武器尺寸完全一致，避免出现“下方是小图标，拖起来突然变大”的割裂感。

## 设计基准

- 游戏逻辑分辨率固定为 `720 x 1440`。
- 当前背包单格尺寸为 `120 x 120`。
- 背包格子间距为 `5`。
- 下方备战区和上方背包使用同一套单格尺寸。
- 武器在备战区、拖拽中、背包内的占格尺寸应保持一致。

## 占格尺寸表

多格物品的图片包围盒按下面公式计算：

```text
宽度 = 格子列数 * 120 + (格子列数 - 1) * 5
高度 = 格子行数 * 120 + (格子行数 - 1) * 5
```

常用尺寸：

| 形状 | 表 id | 占格 | 图片目标包围盒 |
| --- | --- | --- | --- |
| 单格 | `single` | `1 x 1` | `120 x 120` |
| 横二 | `h2` | `2 x 1` | `245 x 120` |
| 竖二 | `v2` | `1 x 2` | `120 x 245` |
| L 型 | `l3` | `2 x 2` | `245 x 245` |
| T 型 | `t4` | `3 x 2` | `370 x 245` |
| 横三 | `h3` | `3 x 1` | `370 x 120` |

## 当前种菜皮肤

当前 demo 是种菜表现，可以按“每格一株/一段”来做：

- 小麦是 `v2`，显示为上下两格，两株小麦。
- 胡萝卜是 `h2`，显示为左右两格，两根胡萝卜。
- 单格番茄、蓝莓等显示为 `1 x 1`。

这种皮肤可以复用一张单格作物图，由程序按形状格子重复摆放。美术只需要保证单格作物在 `120 x 120` 的视觉范围里好看。

建议单格作物源图：

- 推荐源图尺寸：`256 x 256` 或 `512 x 512`。
- 透明背景 PNG/WebP。
- 主体可以适当超出格子一点，但不要超过目标包围盒边缘太多。
- 底部泥土或阴影最好落在格子底部区域，便于多格物品底部对齐。

## 后续武器皮肤

如果后续换成参考视频那种武器表现，不一定需要“每格一张图”。同一个多格物品可以只用一张完整武器图：

- `h2` 武器可以是一张 `245 x 120` 的横向完整图。
- `v2` 武器可以是一张 `120 x 245` 的竖向完整图。
- `h3` 武器可以是一张 `370 x 120` 的长武器图。
- `l3`、`t4` 可以使用对应包围盒的完整异形构图。

换成完整武器图时，图片本身应该已经适配目标包围盒，不要再强行按格重复。程序侧需要支持两种显示模式：

- `tile`：当前种菜皮肤，按每个格子重复/拼接显示。
- `singleArt`：后续武器皮肤，一张完整图适配整个多格包围盒。

## 出图建议

- 每个武器资源优先按“目标包围盒”制作透明图。
- 主体不要贴边，建议四周保留 `6-12` 像素安全边距。
- 需要有清晰外轮廓，适合在绿色背景、背包格子和战斗底部栏中识别。
- 不要把品质边框、CD 遮罩、选中框画进武器图，品质和状态由程序叠加。
- 同一武器不同品质可以复用同一张图，颜色/品质边框由程序处理；如果品质需要明显换外观，再增加独立资源 key。

## 配表入口

武器形状：

```text
public/gamedata/s_item_shape.json
```

武器数据和图片 key：

```text
public/gamedata/s_item.json
```

图片资源路径：

```text
public/gamedata/s_asset.json
public/game-assets/weapons/
```

当前阶段如果只是替换图片，优先保持 `s_item.json` 的 `iconAssetKey` 不变，只改 `s_asset.json` 里的 `url`。
