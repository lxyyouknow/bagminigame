# 小僵尸尺寸与出生间距设计

## 目标

- 将 `zombie_walk_down` 的显示缩放从 `0.26` 调整为 `0.34`，让小僵尸在战斗画面中更清晰。
- 普通波次优先让怪物横向分散出生，避免多个小僵尸刷新时挤在一起。
- 后期怪物数量超过出生空间容量时允许靠近，不能因为间距要求阻塞刷怪。

## 配置

小僵尸大小继续由 `public/gamedata/s_animation.json` 的 `zombie_walk_down.scale` 控制，本次设为 `0.34`。后续只需调整该字段，不修改战斗代码。

## 出生规则

战斗顶部安全范围内预设 6 个横向出生槽位。每次刷怪时：

1. 计算每个槽位与当前存活怪物的最近横向距离。
2. 优先选择最近距离最大的槽位，让新怪尽量远离现有怪物。
3. 当场上怪物较多、所有槽位都已接近怪物时，仍选择相对最疏的槽位，并通过小幅纵向偏移降低完全重叠的概率。
4. 不改变波次配置、刷怪数量和刷怪时间。

## 代码边界

- 新增纯规则函数负责计算出生坐标，便于独立测试。
- `BattleScene` 只负责提供屏幕宽度、当前怪物坐标和随机数，并使用规则函数返回的坐标。
- 不加入怪物之间的实时碰撞推挤，避免影响移动速度和战斗数值。

## 验证

- 单元测试覆盖低密度时优先选择最远槽位，以及高密度时仍能返回有效坐标。
- 配置测试确认小僵尸缩放为 `0.34`。
- 浏览器实际进入第一波，确认小僵尸尺寸和出生分布。
