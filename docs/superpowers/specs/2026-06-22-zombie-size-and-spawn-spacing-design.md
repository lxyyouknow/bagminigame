# 小僵尸尺寸、出生间距与战斗动画暂停设计

## 目标

- 将 `zombie_walk_down` 的显示缩放从 `0.26` 调整为 `0.34`，让小僵尸在战斗画面中更清晰。
- 普通波次优先让怪物横向分散出生，避免多个小僵尸刷新时挤在一起。
- 后期怪物数量超过出生空间容量时允许靠近，不能因为间距要求阻塞刷怪。
- 暂停战斗时同步暂停主角、怪物和战斗特效帧动画，继续战斗时统一恢复。

## 配置

小僵尸大小继续由 `public/gamedata/s_animation.json` 的 `zombie_walk_down.scale` 控制，本次设为 `0.34`。后续只需调整该字段，不修改战斗代码。

小僵尸移动速度继续由 `public/gamedata/s_monster.json` 中 `monsterId: 1` 的 `speed` 控制，目前保持为 `42`。本次不修改移动速度数值。

## 出生规则

战斗顶部安全范围内预设 6 个横向出生槽位。每次刷怪时：

1. 计算每个槽位与当前存活怪物的最近横向距离。
2. 优先选择最近距离最大的槽位，让新怪尽量远离现有怪物。
3. 当场上怪物较多、所有槽位都已接近怪物时，仍选择相对最疏的槽位，并通过小幅纵向偏移降低完全重叠的概率。
4. 不改变波次配置、刷怪数量和刷怪时间。

## 代码边界

- 新增纯规则函数负责计算出生坐标，便于独立测试。
- `BattleScene` 只负责提供屏幕宽度、当前怪物坐标和随机数，并使用规则函数返回的坐标。
- 不加入怪物之间的实时碰撞推挤，避免影响移动速度和战斗数值。

## 战斗动画暂停

`BattleScene` 新增统一的 `setBattlePaused(paused)` 入口，所有暂停、继续、肉鸽选择、波次结算和战斗结算状态都通过该入口修改暂停状态，禁止继续直接写 `this.paused`。

动画控制使用通用工具函数递归遍历指定战斗显示层中的 `AnimatedSprite`：

- 暂停时调用 `stop()`，冻结主角、小僵尸和后续加入的战斗帧动画。
- 继续时只恢复暂停前正在播放的动画，避免误启动原本处于停止状态的一次性动画。
- 暂停弹窗和设置窗口不放在战斗动画根节点内，因此保持正常交互和表现。
- 新创建的战斗动画仍按现有流程播放；战斗暂停期间逻辑不更新，不会继续刷怪或创建攻击特效。

## 验证

- 单元测试覆盖低密度时优先选择最远槽位，以及高密度时仍能返回有效坐标。
- 配置测试确认小僵尸缩放为 `0.34`。
- 单元测试覆盖暂停时停止正在播放的动画、继续时恢复，以及不误启动原本停止的动画。
- 浏览器实际进入第一波，确认小僵尸尺寸和出生分布。
- 浏览器打开暂停弹窗，确认主角和小僵尸帧动画静止；继续后恢复播放。
