# 友方基地与 Boss 动作视频方案

本文记录近期新增概念资源的后续动作规划和视频生成提示词。后续这类资源默认走：

```text
角色概念图
-> 动作视频生成
-> 导出视频帧或 TexturePacker 图集
-> 导入脚本整理为 frames/sheet/gif/metadata
-> 配置 s_asset.json / s_animation.json / s_monster.json 或后续 actor/base 视觉表
```

不默认绑定某个生成工具。普通小怪、简单特效可以使用专门的序列帧工具；Boss、主角、基地炮台、复杂动作优先用视频生成或手动分镜方案。

## 通用视频生成规范

### 画面与方向

- 竖屏 H5 休闲塔防小游戏风格。
- Q 版卡通，俯视略带正面角度。
- 友方基地单位必须表现为玩家方：朝画面上方、朝中心发射器，或左右侧向工作。
- 怪物和 Boss 可以朝画面下方移动，符合敌人从上往下进攻的方向。
- 单个动作视频只包含一个动作，不要把 idle、attack、death 混在同一个视频里。
- 每个视频里主体只能出现一次，不要出现两个角色、残影分身、UI、文字、血条或水印。

### 背景色

默认使用纯洋红背景：

```text
#FF00FF
```

原因：

- 当前资源大量使用绿色：植物炮台、藤蔓、叶子、毒蝠、毒雾、蘑菇配饰。
- 如果使用绿幕 `#00FF00`，容易把绿色主体、边缘光效或叶片误抠掉。
- 洋红在当前田园资源里很少出现，更适合自动抠主体。

要求：

- 背景必须是完全纯色 `#FF00FF`。
- 不要渐变、阴影、地面、环境光、接触阴影、反射或纹理。
- 主体内部不要使用接近 `#FF00FF` 的颜色。
- 主体四周留足安全边距，避免视频生成时动作切边。

特殊情况：

- 如果某个角色主体确实有大量洋红、粉紫、紫红元素，再改用纯蓝 `#0000FF` 或纯绿 `#00FF00`。
- 绿色植物、毒雾、毒蝠、藤蔓类资源不要优先用绿幕。

### 帧率和时长

推荐视频生成参数：

- `idle`：0.8 到 1.2 秒，8 到 12 帧，循环。
- `work_loop`：0.8 到 1.2 秒，8 到 12 帧，循环。
- `attack` / `press` / `pull` / `load`：0.45 到 0.75 秒，6 到 10 帧，非循环。
- `hit`：0.25 到 0.45 秒，4 到 6 帧，非循环。
- `death`：0.8 到 1.4 秒，10 到 18 帧，非循环。

导入到项目后，以 `s_animation.json` 的 `fps` 和 `loop` 为准。

### 导入后目录

友方基地和助手建议：

```text
public/game-assets/characters/<anim-key>/
public/game-assets/base/<anim-key>/
```

Boss 和怪物建议：

```text
public/game-assets/enemies/<anim-key>/
```

源视频或源图集归档：

```text
public/game-assets/source/<asset-name>-video-<timestamp>/
```

## 资源一：植物炮台生命体

定位：替代底部站桩主角的核心发射装置。它是友方植物炮台，不是怪物。

建议动画 key：

- `plant_turret_idle`
- `plant_turret_attack`
- `plant_turret_load`，可选
- `plant_turret_hit`，可选

动作设计：

- `idle`：藤蔓臂轻轻摆动，叶子托盘轻微呼吸，底座表情眨眼。
- `attack`：藤蔓臂后拉，托盘下压，果实从托盘向上弹出，藤蔓回弹。
- `load`：叶子托盘轻轻打开或抬起，准备放入当前武器果实。
- `hit`：炮台受击时叶子抖动、底座皱眉，可先不做。

重要拆层建议：

- 炮台本体动画里不要画死具体武器果实。
- 托盘上方保留一个空位，运行时单独叠加当前 CD 好的武器果实。
- 如果视频生成必须带示例果实，只能用简单番茄占位，后续导入时尽量拆掉或不使用该层。

视频生成提示词：

```text
生成 2D 动作视频，纯 #FF00FF 背景，单个友方植物炮台生命体，Q版卡通，俯视略带正面角度，朝画面上方发射，适合竖屏 H5 休闲塔防小游戏。

角色是玩家方的活体植物炮台，不是怪物。主体由南瓜或木桩底座、两条藤蔓弹弓臂、绿色叶子托盘组成。底座有友好表情，藤蔓像弹弓一样可以后拉。整体可爱、可靠、农场主题，轮廓清晰，颜色块明确，适合 720x1440 手机画面识别。

动作：attack。藤蔓臂先向后拉，叶子托盘轻微下压蓄力，然后向上释放发射，最后藤蔓回弹到原位。动作要清楚但不要大幅位移。炮台主体中心点保持稳定，不要漂移，不要旋转成侧面。不要出现人物、敌人、文字、UI、血条、复杂背景、地面阴影或多个炮台。

背景必须是完全纯色 #FF00FF，无阴影、无渐变、无纹理。主体内部不要使用 #FF00FF。视频 0.6 秒左右，8 帧左右，非循环，便于切成 plant_turret_attack 序列帧。
```

`idle` 变体提示词：

```text
动作改为 idle。藤蔓臂轻轻摇摆，叶子托盘轻微呼吸起伏，底座眨眼。动作幅度小，可无缝循环，视频 1 秒左右，10 到 12 帧，loop=true。
```

## 资源二：小动物后勤队

定位：基地左右两侧的友方小帮手，不直接作为主攻击角色。

建议动画 key：

- `mole_loader_idle`
- `mole_loader_load`
- `chick_operator_idle`
- `chick_operator_pull`

动作设计：

- 鼹鼠装填手：搬番茄、推小篮筐、把果实递向中间托盘。
- 小鸡操作手：拉绳、按按钮、被发射反作用力轻轻弹一下。
- 两个角色应保持小体型，不要比植物炮台更抢眼。

视频生成提示词，小鼹鼠装填手：

```text
生成 2D 动作视频，纯 #FF00FF 背景，单个友方小鼹鼠装填手，Q版卡通，俯视略带正面角度，玩家方基地助手，朝画面上方或朝画面中心侧向工作，适合竖屏 H5 休闲塔防小游戏。

小鼹鼠是可爱的后勤角色，不是怪物。圆润棕色身体，浅色鼻子和小爪子，戴小农夫帽或绿色叶子围巾，表情认真友好。它抱着一个番茄或胡萝卜弹药，向画面中心的发射托盘方向递送。不要画完整投石车，不要画巨大推车，只保留小篮筐或单个果实。

动作：load。小鼹鼠从准备姿势开始，双手抱起果实，向中心方向递出或轻轻放下，再回到准备姿势。身体中心点稳定，动作清楚，不能出现两个鼹鼠或残影分身。不要文字、UI、血条、复杂背景、敌人表情。

背景必须是完全纯色 #FF00FF，无阴影、无渐变、无纹理。主体内部不要使用 #FF00FF。视频 0.6 秒左右，8 帧左右，非循环，便于切成 mole_loader_load 序列帧。
```

视频生成提示词，小鸡操作手：

```text

小鸡是可爱的后勤角色，不是怪物。圆润黄色小鸡，戴小叶子头巾或小围裙，表情认真可爱。旁边只有一个小木制拉杆、绳子或按钮，不要出现完整投石车。小鸡负责拉动发射机关。

动作：pull。小鸡双翅抓住绳子或拉杆，先用力后拉，身体轻微后仰，然后松开并被反作用力小小弹回，最后回到准备姿势。动作方向要读成友方从底部操作发射器，不要像敌人向下攻击。不要文字、UI、血条、复杂背景、敌人表情。

背景必须是完全纯色 #FF00FF，无阴影、无渐变、无纹理。主体内部不要使用 #FF00FF。视频 0.6 秒左右，8 帧左右，非循环，便于切成 chick_operator_pull 序列帧。
```

## 资源四：腐藤南瓜王 Boss

定位：田园主题最终波 Boss。怪物方向朝画面下方。

建议动画 key：

- `pumpkin_king_walk_down`
- `pumpkin_king_attack_down`
- `pumpkin_king_hit`
- `pumpkin_king_death`

动作设计：

- `walk_down`：沉重慢走，身体上下压缩，藤蔓手臂拖动。
- `attack_down`：藤蔓手臂抬起，向下拍击基地方向。
- `hit`：身体轻震，南瓜壳裂纹短暂亮起。
- `death`：南瓜壳裂开，藤蔓枯萎，泥土和叶片散开，最后塌成一团。

视频生成提示词：

```text

Boss 是腐藤南瓜王，巨大腐化南瓜和泥土藤蔓融合的田园怪物。厚重南瓜身体，短粗根须脚，藤蔓手臂，枯叶和少量小蘑菇点缀，眼睛淡黄绿色发光。它是敌方最终波核心怪，血量高、护甲高、攻击高、速度慢。风格可爱但有压迫感，不恐怖不血腥。

动作：walk_down。Boss 缓慢朝画面下方迈步，身体沉重上下晃动，短脚交替挪动，藤蔓手臂轻轻拖动。动作幅度稳，不要跳跃，不要旋转方向，不要出现第二个 Boss。轮廓清晰，颜色块明确，适合手机画面识别。

背景必须是完全纯色 #FF00FF，无阴影、无渐变、无纹理。主体内部不要使用 #FF00FF。视频 1 秒左右，10 到 12 帧，可循环，便于切成 pumpkin_king_walk_down 序列帧。
```


## 后续接入检查清单

每个动作视频生成后，先人工检查：

- 主体方向是否符合阵营：友方朝上或侧向中心，敌方朝下。
- 是否有纯色背景，边缘是否方便抠图。
- 是否只有一个主体，没有残影分身。
- 动作是否能读懂，首尾是否适合循环或非循环播放。
- 主体是否一直在安全框内，没有切边。

导入后检查：

```bash
npm run test:monster-config
npm run test:monster-visual
npm run test:skill-visual
node -e "const fs=require('fs'); for (const f of fs.readdirSync('public/gamedata').filter(f=>f.endsWith('.json'))) JSON.parse(fs.readFileSync('public/gamedata/'+f,'utf8')); console.log('json ok')"
npm run build
```

如果是友方基地或助手动作，后续新增对应测试脚本后，也要加入专门的 `base/actor visual` 配置校验。
